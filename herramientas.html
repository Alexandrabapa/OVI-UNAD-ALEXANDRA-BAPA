GITHUB





¿Que es?

Git hub es una plataforma que nos permite crear repositorios, de nuestros softwares creados o de cualquier otro que se quiera subir para que otras personas den su opinión, y tienen permisos para modificarlos. Todo depende de los permisos que el administrador o usuario que suba el software le dé a otros usuarios.

Una vez dentro de la página principal de Git Hub se tiene la opción de registrarse llenando un formulario de inscripción. Una vez dentro se tendrá acceso a los repositorios creados



¿como funciona?



Con Git Hub podemos dar seguimiento a todos los cambios de los proyectos,

¿pero, como funciona?

La carpeta con todas las copias de tu proyecto en términos de Versión Control se llama repositorio, es una colección de todas las versiones de un proyecto en particular junto con otra información adicional, como en qué orden transcurrieron los cambios, una descripción de cada cambio y una que otra información como quien es el responsable de cada cambio. Cada proyecto debe tener su propio repositorio, un VCS no hace seguimiento a las versiones de los proyectos automáticamente, hay que especificarle cuando una versión está terminada, cuando sucede esto recibe el nombre de Committing.

Un VCS guarda toda la información en carpetas especiales, las cuales están ocultan en cada proyecto, Para tener acceso a las versiones anteriores se necesita una terminal o consola. Un VCS te permite ver y filtrar la lista completa de Cummings o cambiar la versión que el folder de tu proyecto actualmente está mostrando, también nos da la forma de compartir los repositorios con otros, es así como se puede colaborar en otros proyectos y dar seguimiento a los cambios que otros hacen.

El VCS sobre el cual vamos a prender es GIT, pues la manera en como el está diseñado hace que sea una buena opción, ya que se caracteriza principalmente por ser un VCS distribuido, algunos VCS son centralizados es decir que solo hay un repositorio guardado generalmente en un servidor, si se desea trabajar sobre ese proyecto se debe hacer sobre una red a ese repositorio central, esto tiene algunas desventajas pue hace difícil trabajar sobre el proyecto sin una conexión de red.

Si se llega a presentar un daño en el repositorio central, se puede perder alguna información o incluso todo el historial del proyecto La colaboración se hace más restringida y complicada Un VCS distribuido como GIT son diferentes no existe un repositorio central todos los que trabajan en un proyecto tienen su propio repositorio, las contribuciones al proyecto no requieren conexión a internet, esto hace que todo sea más rápido.
